# Changelog

All notable changes to this project will be documented in this file.

## [0.1.0] — Foundation release

### Added

- AI accountability design-pattern catalog covering governance, transparency, human oversight, and redress patterns.
- Accountability matrix template for named ownership, decision rights, oversight, escalation, redress, and audit trail requirements.
- Filled generic sample accountability matrix.
- NIST AI RMF mapping references for accountability-related patterns.
- README guidance clarifying the relationship between this repo and adjacent governance, release, and validation repositories.

### Notes

This repository focuses on accountability design: who owns decisions, how escalation works, and how affected users or operators can challenge AI-assisted outcomes.

### Next

- Add pattern-specific examples for override mechanisms, appeal pathways, adverse-action explanations, and decision logs.
- Add a machine-readable accountability review schema.
- Add more examples for medium-risk and high-risk AI systems.
